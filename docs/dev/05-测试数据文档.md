# JudgeMesh 测试数据文档

本文档用于统一 JudgeMesh 在本地开发、联调、自测、答辩演示中的测试数据口径。

## 1. 文档目标

- 为用户、题目、竞赛、提交、判题链路提供一套可重复使用的测试数据。
- 覆盖正常路径、异常路径、高并发路径和分布式判题路径。
- 让前端展示、接口联调、压测脚本、答辩演示使用同一批基础数据。

## 2. 设计原则

| 项目 | 说明 |
| --- | --- |
| 真实性 | 数据规模和字段风格接近真实 OJ 平台，而不是只放几条占位记录。 |
| 可验证性 | 每条关键测试数据都能对应明确的预期结果，例如 AC、WA、RE、TLE、SE。 |
| 可复用性 | 同一批账号和题目既能用于手工测试，也能用于脚本压测。 |
| 可扩展性 | 预留批量用户、批量提交、竞赛扩容空间，便于后续继续加数据。 |
| 可演示性 | 包含足够直观的成功案例和失败案例，便于前端页面与答辩讲解。 |

## 3. 测试范围概览

| 维度 | 覆盖内容 | 主要用途 |
| --- | --- | --- |
| 用户与权限 | 管理员、出题人、学生、游客 | 登录、RBAC、余额扣减、页面权限控制 |
| 题目与测试点 | 简单题、中等题、异常题、缺失数据题 | 判题正确性、错误分支验证 |

| 高并发压测 | 批量账号、批量提交 | 脚本压测、消息队列、自动扩缩容验证 |

## 14. 使用建议

- 联调阶段优先使用第 4、5、6、7 节的数据。
- 答辩演示优先使用第 8、9、10、11 节的数据。
- 如果当前集群资源紧张，先使用 `3 worker + 30 用户 + 30 提交` 的轻载数据。
- 如果要做高并发展示，再额外准备 `60` 或 `120` 个压测账号。
